refresh

return type undefined
syntax .geomap( "refresh" [ , force ] )
usage 
$("#map").geomap( "refresh" )

$("#map").geomap( "refresh", true )

This method refreshes all services and graphics in the map. Usually, changing geomap options or calling methods automatically refreshes the map appropriately. However, you can call this after you have added or removed shapes to geomap by passing false as the refresh argument to the append, remove, or empty methods.

The force option will re-request images for tiles already in the map view. Keep in mind, though, that this does not guarantee you will get updated images. See the Dynamic Data section below for tips.

Dynamic Data

There are three factors that can prevent jQuery Geo from getting a fresh image from a web server.

  • if the user does not interact with the map, jQuery Geo will not request a new image or new tiles
  • if the user does interact with the map, you call refresh with no arguments, or you call .geomap( "refresh", false ), jQuery Geo will only request images for tiles that it does not already have for tiled services; a new image is always requested for shingeld service
  • if you call .geomap( "refresh", true ), jQuery Geo will request a new image for existing tiles but the web browser is free to supply a cached image instead of retriving one from the web server; this is also true for shinged services if the final image URL is the same as a previous image URL

A web browser will normally cache images that have already been loaded. On top of that, jQuery Geo will not, by default, request images for tiles that it has already placed. In the case of dynamic data, where you want to update what's on the map without requiring a pan or zoom, you have a couple options.
