addShape

return type undefined
syntax .geomap( "addShape" , Object ( GeoJSON object ) [ , Object ( geomap style ) ] )
usage 
$("map").geomap( "addShape", { type: "Point", coordinates: [ -71, 40 ] } )
$("map").geomap( "addShape", { type: "Point", coordinates: [ -71, 40 ] }, { width: "16px", height: "16px" } )

The addShape method adds a GeoJSON geometry, feature or feature collection to the map. Only basic geometry types are supported, even for features. For example, you can add a GeoJSON Point, LineString, Polygon, a Feature with one of those three geometry types or a FeatureCollection where all the features are one of those three geometry types.

Unsupported geometry types are ignored so you can call addShape with a FeatureCollection that has some MultiLineString geometry mixed in. The MultiLineString features will not be added, the others will.

If $.geo.proj is not null, all coordinates of all geometry (whether plain or part of a feature) must be in as geodetic coordinates, e.g., for a Point object coordinates[0] is longitude and coordinates[1] is latitude. Otherwise, all coordinates must be in the widget's internal projection as defined by tilingScheme for tiled map services or bbox for shingled (dynamic) map services.

The optional style argument modifies how geomap draws the geometry you just added. This style will override the style set on the geomap widget via shapeStyle method calls. Please see the shapeStyle method documentation for information about what properties are valid for this object.
